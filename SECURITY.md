# Security Policy

## 支持版本

当前仓库以 `main` 分支最新代码为主要维护对象。历史版本不保证安全更新。

## 报告安全问题

如果你发现以下问题，请不要公开提交 Issue：

- 凭证泄露（Cookie / Token / 密钥）
- 可被利用的接口缺陷
- 任意代码执行、信息泄露、权限绕过等漏洞

请通过私下渠道联系维护者，并尽量提供：

- 漏洞描述与影响范围
- 复现步骤
- 可能的修复建议

## 响应流程（目标）

- 24-72 小时内确认收到报告
- 7 天内给出初步评估
- 在可行范围内尽快发布修复

## 安全实践建议

- 不要提交任何真实 `credentials.json`
- 建议定期更换 Cookie，避免长期复用
- 提交前执行：
  - `git status`
  - 检查是否误加入本地数据与日志文件
