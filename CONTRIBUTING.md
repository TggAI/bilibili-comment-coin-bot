# Contributing

感谢你愿意参与这个项目。

## 提交规范

- 使用清晰的提交信息，说明变更目的
- 一个 PR 聚焦一类改动（修复、功能、重构分开）
- 避免提交本地数据、凭证和日志文件

## 开发建议

1. Fork 并新建分支：
   - `feat/xxx`
   - `fix/xxx`
2. 本地安装依赖并验证基础运行：
   - `python main.py BVxxxxxxxxxx`
   - `python server.py`
3. 提交 PR，描述：
   - 改了什么
   - 为什么改
   - 如何验证

## 安全相关

- 不要提交任何真实 Cookie、Token、账号信息
- 若发现敏感信息泄露风险，请先私下联系维护者处理
